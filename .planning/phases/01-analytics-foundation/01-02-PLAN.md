  <name>Task 1: Create pure-function backtester with historical entry re-evaluation</name>
  <files>
    src/domain/backtester.js

  <action>
    **1. Create `src/domain/backtester.js`:**

    This is a pure-function module (no I/O, no imports except maybe from config for defaults). It must follow the existing domain layer pattern (see `entryGate.js`, `exitEvaluator.js`, `sizing.js` -- all pure functions).

    **Export `evaluateHistoricalEntry(trade, config)`:**
    - Re-evaluates whether a historical trade would have passed entry gate thresholds with the given config overrides.
    - Only checks configurable threshold-based blockers (NOT time-dependent or state-dependent ones like "trading disabled", "market closed", "already in position"). The user decision specifies this enables near-miss analysis for threshold tuning.
    - Threshold checks to re-evaluate (using `*AtEntry` fields from enriched trades):
      - `modelProbAtEntry` vs `config.minProbMid` (probability threshold)
      - `edgeAtEntry` vs `config.edgeMid` (edge threshold)
      - `rsiAtEntry` vs `config.noTradeRsiMin` / `config.noTradeRsiMax` (RSI no-trade band)
      - `spreadAtEntry` vs `config.maxSpreadThreshold` (max spread)
      - `liquidityAtEntry` vs `config.minLiquidity` (min liquidity)
      - `spotImpulsePctAtEntry` vs `config.minSpotImpulse` (min impulse)
    - For each threshold, if the trade's `*AtEntry` field is null/undefined/not-a-number, SKIP that check (treat as "unknown, don't filter"). This handles historical trades pre-enrichment.
    - Return `true` if the trade would have entered, `false` if filtered out.
    - CRITICAL: Never use exit-time data (`exitPrice`, `exitReason`, `pnl`) to decide entry. This prevents look-ahead bias (see RESEARCH.md Pitfall 3).

    **Export `replayTrades(trades, overrideConfig, baseConfig)`:**
    - Accepts array of trade objects, override params, and base config.
    - Merges config: `const config = { ...baseConfig, ...overrideConfig }`
    - Filters to CLOSED trades only.
    - For each closed trade, calls `evaluateHistoricalEntry(trade, config)`.
    - Tracks: `entered[]`, `filtered[]`, `totalPnl`, `wins`, `losses`.
    - Computes derived metrics:
      - `winRate`: wins / entered.length (null if no entered trades)
      - `profitFactor`: sum(winning PnL) / abs(sum(losing PnL)) (null if no losses)
      - `maxDrawdown`: compute max drawdown from entered trades (reuse the drawdown logic pattern from analyticsService -- copy the pure function, do not import from analyticsService to keep domain layer pure)
      - `avgWin`, `avgLoss`, `expectancy`
    - Returns `{ entered, filtered, totalPnl, wins, losses, winRate, profitFactor, maxDrawdown, avgWin, avgLoss, expectancy, tradeCount: entered.length, filteredCount: filtered.length }`.

    **2. Create `test/domain/backtester.test.js`:**

    Test `evaluateHistoricalEntry()`:
    - Trade with all fields above thresholds -> returns true
    - Trade with prob below minProbMid -> returns false
    - Trade with edge below edgeMid -> returns false
    - Trade with RSI in no-trade band -> returns false
    - Trade with null enrichment fields (historical) -> returns true (unknown = don't filter)
    - Trade with some fields null, some failing -> correctly filters on available data only

    Test `replayTrades()`:
    - 10 mock trades with known PnL values and enrichment fields
    - Override minProbMid higher -> fewer trades entered, different win rate
    - Override edgeMid higher -> fewer trades entered
    - All trades filtered -> returns null metrics gracefully
    - Empty trades array -> returns empty results gracefully
    - Verify profitFactor, winRate, maxDrawdown computations against hand-calculated values

    **1. Create `src/services/backtestService.js`:**

    This is the application/service layer that connects the pure backtester to I/O (ledger data, config).

    **Export `runBacktest(overrideConfig)`:**
    - Loads trades from the paper ledger: `initializeLedger()` then `getLedger().trades`
    - Gets base config from `CONFIG` (import from `../config.js`)
    - Calls `replayTrades(trades, overrideConfig, baseConfig)` from `../domain/backtester.js`
    - Returns the replay result object, enriched with:
      - `baseConfig`: the relevant threshold values from base config (so UI can show "current vs tested")
      - `overrideConfig`: what was overridden
      - `totalTradesInLedger`: total trades in ledger (for context)
      - `enrichedTradeCount`: count of trades that have non-null enrichment fields (to show data quality)
    - Handle errors gracefully: if ledger not initialized, return error object.

    **2. Add `POST /api/backtest` endpoint to `src/ui/server.js`:**

    Add after the existing analytics endpoint:
    ```
    POST /api/backtest
    Body: { params: { minProbMid: 0.55, edgeMid: 0.04, ... } }
    Response: { success: true, data: { ...replayResult, baseConfig, overrideConfig } }
    ```
    - Import `runBacktest` from `../services/backtestService.js`
    - Validate `req.body.params` exists and is an object
    - Whitelist allowed override keys to prevent injection: only `minProbMid`, `edgeMid`, `noTradeRsiMin`, `noTradeRsiMax`, `maxSpreadThreshold`, `minLiquidity`, `minSpotImpulse`, `maxEntryPolyPrice`
    - Filter out any keys not in the whitelist
    - Validate all values are numbers
    - Call `runBacktest(validatedParams)`
    - Return result wrapped in `ok()` helper

    **Error handling:**
    - Missing/invalid body -> 400 with descriptive message
    - Empty params -> run backtest with base config (effectively shows current performance)
    - Backtester throws -> 500 with error message

    Do NOT add the optimizer endpoint yet (that is Plan 03). Do NOT add config apply endpoints yet (Plan 03).
