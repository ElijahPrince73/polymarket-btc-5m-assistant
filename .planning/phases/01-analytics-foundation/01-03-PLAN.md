  <action>
    **1. Create `src/domain/optimizer.js`:**

    Pure-function module (no I/O). Follows existing domain layer pattern.

    **Export `cartesianProduct(paramRanges)`:**
    - Input: `{ paramName: [val1, val2, ...], ... }`
    - Output: array of arrays, each sub-array is one combination
    - Iterative implementation (not recursive) to avoid stack overflow on large grids

    **Export `gridSearch(trades, baseConfig, paramRanges, minTradesPerCombo = 30)`:**
    - Computes all combinations via `cartesianProduct(paramRanges)`
    - For each combination, builds overrideConfig, calls `replayTrades(trades, overrideConfig, baseConfig)` from `../domain/backtester.js`
    - Skips combinations where `replay.entered.length < minTradesPerCombo` (per user decision: minimum 30 trades prevents overfitting)
    - Collects results: `{ params, tradeCount, filteredCount, winRate, profitFactor, totalPnl, maxDrawdown, avgWin, avgLoss, expectancy }`
    - Sorts by profitFactor descending (primary), winRate descending (secondary)
    - Returns `{ results, totalCombinations, skippedCombinations, paramNames }`
    - If total combinations exceeds 10,000, throw an error with a descriptive message suggesting fewer params or coarser steps (per RESEARCH.md Pitfall 4)

    **Export `generateParamRanges(rangeConfig)`:**
    - Helper to convert `{ paramName: { min, max, step } }` into `{ paramName: [val1, val2, ...] }` arrays
    - Handles floating point step sizes correctly (avoid accumulation errors by using integer multiples)

    **Default param ranges** (per RESEARCH.md suggestions, Claude's discretion):
    ```
    DEFAULT_PARAM_RANGES = {
      minProbMid: { min: 0.50, max: 0.58, step: 0.01 },
      edgeMid: { min: 0.01, max: 0.06, step: 0.01 },
      noTradeRsiMin: { min: 25, max: 40, step: 5 },
      noTradeRsiMax: { min: 40, max: 55, step: 5 },
      maxEntryPolyPrice: { min: 0.004, max: 0.008, step: 0.001 },
    }
    ```
    Export as `DEFAULT_PARAM_RANGES` so the UI can display it.

    **2. Add API endpoints to `src/ui/server.js`:**

    **POST /api/optimizer:**
    - Body: `{ paramRanges?: { paramName: { min, max, step } }, minTrades?: number }`
    - If `paramRanges` not provided, use `DEFAULT_PARAM_RANGES`
    - Load trades from paper ledger
    - Call `generateParamRanges()` then `gridSearch()`
    - Return `ok({ results, totalCombinations, skippedCombinations, paramNames, currentConfig })` where currentConfig is the relevant threshold values from the running engine config
    - Wrap in try/catch; if grid too large (>10K combos), return 400 with error

    **POST /api/config:**
    - Body: `{ params: { minProbMid: 0.55, ... } }`
    - Per user decision: updates running config with selected parameter set, changes take effect on next engine tick
    - Store previous config for revert: `globalThis.__previousConfig = { ...relevantFields }`
    - Whitelist allowed keys (same as backtest endpoint)
    - Apply to `engine.config` via `Object.assign`
    - SAFETY per RESEARCH.md Pitfall 5: If live mode is active (`engine.executor.getMode() === 'live'`), include a warning in the response: `{ applied: params, revertAvailable: true, warning: 'Live mode active -- parameters affect real trading' }`
    - Return `ok({ applied, revertAvailable: true, warning? })`

    **POST /api/config/revert:**
    - Restores `globalThis.__previousConfig` to engine.config
    - Returns `ok({ reverted: true })` or 400 if no previous config

    **GET /api/config/current:**
    - Returns current threshold config values (the whitelisted subset) so the UI can show current values

    **3. Create `test/domain/optimizer.test.js`:**

    - Empty input -> single empty combination

    Test `generateParamRanges`:
    - Known range config -> correct array of values
    - Verify floating point correctness (e.g., 0.01 steps from 0.50 to 0.53 = [0.50, 0.51, 0.52, 0.53])

    Test `gridSearch`:
    - Mock 50 trades with enrichment fields
    - Small param ranges (2x2 = 4 combos)
    - Verify results sorted by profitFactor descending
    - Verify combinations with <minTradesPerCombo are skipped
    - Verify error thrown when combos > 10,000
