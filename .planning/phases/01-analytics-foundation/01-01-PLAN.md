    **1. Add period grouping functions to `src/services/analyticsService.js`:**

    Add three key functions (before `computeAnalytics`):

    - `dayKeyFromTrade(trade)` -- Groups by Pacific time day. Use `Intl.DateTimeFormat('en-CA', { timeZone: 'America/Los_Angeles', year: 'numeric', month: '2-digit', day: '2-digit' })` on `trade.exitTime || trade.timestamp`. Return `'unknown'` if no valid timestamp.

    - `weekKeyFromTrade(trade)` -- Groups by ISO week. Format as `'YYYY-Wnn'`. Use the trade's exit time (or timestamp fallback).

    - `sessionKeyFromTrade(trade)` -- Groups by BTC trading session based on UTC hour of entry time: Asia (0-8 UTC), London (8-13 UTC), NY (13-21 UTC), Off-hours (21-0 UTC). Return `'unknown'` if no valid timestamp.

    **2. Add financial metrics functions:**

    - `computeDailyReturns(closedTrades, startingBalance)` -- Groups PnL by day (Pacific time), converts each day's total PnL to a return (dayPnL / startingBalance). Returns array of daily return values.

    - `computeSharpeRatio(dailyReturns, riskFreeRate = 0)` -- Mean excess return / std dev, annualized by sqrt(252). Return null if fewer than 2 data points or if stdDev is 0. Per RESEARCH.md: use daily returns (not per-trade) to avoid inflated ratios.

    - `computeSortinoRatio(dailyReturns, riskFreeRate = 0)` -- Mean excess return / downside deviation, annualized by sqrt(252). Downside deviation computed only from negative returns. Return null if fewer than 2 data points or no negative returns.

    - `computeDrawdownSeries(closedTrades, startingBalance)` -- Returns array of `{ tradeIndex, equity, peak, drawdown, drawdownPct }`. Walk through trades accumulating PnL, track running peak, compute drawdown at each step.

    - `computeMaxDrawdown(closedTrades, startingBalance)` -- Returns `{ maxDrawdownUsd, maxDrawdownPct }` from the drawdown series.

    **3. Extend `computeAnalytics()` return object:**

    Add to the existing return object (do NOT remove any existing fields):
    ```javascript
    // Period groupings
    byDay: groupSummary(closed, dayKeyFromTrade),
    byWeek: groupSummary(closed, weekKeyFromTrade),
    bySession: groupSummary(closed, sessionKeyFromTrade),

    // Advanced metrics
    advancedMetrics: {
      sharpeRatio: computeSharpeRatio(dailyReturns),
      sortinoRatio: computeSortinoRatio(dailyReturns),
      maxDrawdownUsd: maxDD.maxDrawdownUsd,
      maxDrawdownPct: maxDD.maxDrawdownPct,
      drawdownSeries: drawdownSeries,
      dailyReturns: dailyReturns,
      dailyReturnCount: dailyReturns.length,
      metricsConfidence: dailyReturns.length >= 30 ? 'HIGH' : 'LOW',
    }
    ```

    The `startingBalance` for drawdown/returns should come from `CONFIG.startingBalance` (import CONFIG from `../config.js`). If not available, default to 1000.

    **4. Extend existing `groupSummary()` to include win/loss breakdown:**

    Enhance each bucket to include `wins`, `losses`, `winRate`, `avgPnl` alongside existing `count` and `pnl`. This makes period tables immediately useful without client-side computation. Keep backward compatibility -- existing fields unchanged.
